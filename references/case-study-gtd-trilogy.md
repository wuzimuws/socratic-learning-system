## 教学递进轨迹

### 书1：搭系统 → 改造

学生从零开始搭建GTD系统。核心教学动作是**诊断+当场改造**：

```
课前：只有OmniFocus基本使用，无GTD流程
课后：
  - 加持续时间判断字段（当场改造·模式十七扩展四）
  - 拆分明细标签（工作-开发→具体行动标签）
  - 一生价值观文件夹写入第一条（空结构物触发·模式二十一）
  - 9个模糊项目变成具体下一步行动（任务锚型技术·模式二十）
  - 删掉一件挂了不想做的任务（协议释放感·模式十九）

老师轮换：星瑶(导论+第1-3+第11-15章) → 月涵(第2-3+第6+第10+第12-13章) → 霜凝(第7-9+第14章)
```

**核心产出：** "最小化执行目标的意志力消耗，让每个小目标的推进都无比轻松"

### 书2：走原则 → 命名

学生已有系统在跑。教学动作变为**原则贴标签**——每条原则都是对已有实践的确认：

```
诊断：系统在跑、回顾在做、价值观有第一条
教学：52条原则中精选~15条，每条确认"你已经在做了"
现场操作：清掉无下一步行动的项目

检验（霜凝·模式二十三实操书变体）：
  Q1: 暂停项目的判断标准 → "不符合形势、近期无计划" ✅
  Q2: 哪条最不重要 → "这本价值不如第一本" ✅（独立筛选力）
  Q3: 原则与硬截止期冲突 → "该做就做，做不完承认做不完" ✅
```

**核心产出：** "状态差就不做，不内耗" | "该做就做，做不完承认做不完，天塌不下来"

### 书3：画地图 → 对齐

学生已有系统和原则。教学动作变为**框架对齐**——把已有经验拼接到GTD的6层视角×5步掌控十字图上：

```
课前状态：系统顺、9个真实项目、@location标签、每天发视频
教学方式：展示十字图 → 逐层对齐 → 学生自驱修正
  - 横轴5步：捕捉→明确→组织→深思→参与（1遍过，全对齐）
  - 纵轴6层：每条问"你在哪"→学生自报位置
  - 自驱对齐信号：清到9项目、加location标签、翻译目标→行动

框架扩展：
  学生主动问"类似的还可以放什么" → 列出适合5万英尺的类型
  学生自己把分清生物反应和个人认知定位到5万英尺
  学生价值观文件夹写入第二条

检验（三部曲终点站变体）：
  Q1: 框架里有没有没出现的？→ "没有，基本都覆盖了"
  Q2: 哪一层还没到？→ "4万英尺愿景和5万英尺宗旨还需沉淀"
  Q3: 整本留给你的一句话？→ (待检验完成时记录)
```

## 三部曲成功要素分析

### 1. 每本书的教学法不同且不重复

```
书1：诊断+改造 → 工具系统改变
书2：贴标签+确认 → 原则内化
书3：框架对齐+填补 → 经验系统化
```

三部曲教学最忌讳的是用同一方法教三本书。书1的诊断在书2会变成"我都做过了"，书2的原则贴标签在书3会变成"我都知道了"。必须每一本切换老师的角色。

### 2. 自驱对齐现象是三部曲终点站的独有信号

书1的最高信号是"学生当场改了工具"（模式十七扩展四）。
书3的最高信号是"学生在老师还没提要求时自己做了修正"（模式二十八续·扩展一）。

**为什么不同：** 书1改工具是老师引导下的动作。书3改系统是学生自己通过框架诊断盲区、自己做的调整——老师只是提供了一个镜子，学生自己看到了该整理的地方。

### 3. 核心产出逐层积累，不丢失

```
书1 → 书2 → 书3
最小化意志力消耗 → 状态差不内耗 → (放到5万英尺层)
1万粉(价值观第一条) → 每天发视频(下一步行动) → (串起跑道→1万ft→3万ft)
→ 分清生物反应和个人认知(价值观第二条)
```

每条产出都没有被下一本覆盖——而是被下一本的框架吸收了。

### 4. 检验方式逐本变化

```
书1检验（经典三问·模式二十三）：考察是否理解概念
书2检验（实操书变体）：考察原则是否内化到行为判断
书3检验（三部曲终点站变体）：考察框架完整性——有没有盲区
```

如果书3还用书1的检验方式（"告诉我GTD的5步是什么"），学生会觉得无聊。检验的层次必须随着书序递进。

## 可复用的教学模式

```python
三部曲教学总框架：

1. 识别当前是三部曲的第n本
2. 选择对应的教学法：
   n=1 → 模式十七（诊断+改造，从零搭系统）
   n=2 → 模式二十八（贴标签+确认，命名已有实践）
   n=3+ → 模式二十八续（框架对齐+覆盖，经验系统化）
3. 每次切换时，检验方式也升级
4. 学生的核心产出应被下一本的框架吸收而非替换
```
